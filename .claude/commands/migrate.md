Создай и примени миграции для приложения testing.

Описание изменений (если есть): $ARGUMENTS

## Шаги

1. Прочитай `testing/models.py` — убедись что изменения сохранены.
2. Проверь на конфликты миграций:
   ```bash
   python manage.py migrate --check
   ```
3. Создай миграцию:
   ```bash
   python manage.py makemigrations testing
   ```
4. Просмотри созданную миграцию — убедись что она делает именно то, что ожидалось.
5. Примени:
   ```bash
   python manage.py migrate
   ```
6. Финальная проверка:
   ```bash
   python manage.py check
   ```

## Красные флаги — остановись и сообщи пользователю

- Миграция удаляет колонку с данными (`RemoveField`) — уточни, точно ли это нужно.
- Добавляется `NOT NULL`-поле без `default` на существующую таблицу — Django спросит дефолт, предложи разумный.
- Конфликт миграций (две ветки с одним номером) — нужен `python manage.py makemigrations --merge`.
- Переименование поля определяется Django как удаление + добавление — уточни у пользователя намерение перед применением.

## Откат (если нужен)

```bash
python manage.py migrate testing <предыдущий_номер>
```

Номера миграций смотри в `testing/migrations/`.
