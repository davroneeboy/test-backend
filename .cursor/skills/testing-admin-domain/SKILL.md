# Домен админки тестирования

## Сущности (ориентир)

| Концепт | Назначение |
|--------|------------|
| Отдел | Группировка пользователей; в UI — «отдел», в коде — `Group` или `Department` |
| Пользователь | Стандартный `User`; принадлежность к отделу через группу или FK |
| Тест | Набор заданий или параметров экзамена |
| Попытка | Одно прохождение теста пользователем с метками времени |
| Ответ / деталь | При необходимости — ответы по вопросам для разбора |

## Обязательные поля результата попытки

- Связь с пользователем и тестом
- `started_at`, `finished_at` (или одно поле времени + длительность)
- Вычисляемая или сохраняемая **длительность**
- **Итог**: балл, процент, статус (сдан / не сдан / на проверке)

## Админка: что показывать

- Списки попыток: пользователь, тест, отдел (через пользователя), дата, длительность, итог
- Фильтры: по отделу, тесту, диапазону дат
- Для тестов: краткое описание, активен/черновик, дата обновления

## Расширение функционала

1. Добавь или измени модель в `models.py`
2. `makemigrations` / `migrate`
3. Зарегистрируй в `admin.py` с фильтрами и поиском
4. При тяжёлых списках — оптимизация queryset в `ModelAdmin`

Перед выбором «группа Django» vs «модель Отдел» согласуй с существующими миграциями; не дублируй два источника правды.
